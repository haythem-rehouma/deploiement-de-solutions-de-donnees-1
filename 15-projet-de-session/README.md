# Module 15 — Projet de session

Module final du cours **Développement et déploiement de solutions de données** (420-D30-BB).
Il rassemble tout le cours en un projet intégrateur : concevoir et livrer une **chaîne DevOps complète de bout en bout**, du code versionné jusqu'au monitoring en production.

## Objectifs

À la fin de ce module, vous serez capable de :

- **Assembler** en un seul projet toutes les briques du cours : Git, Maven, Docker, CI/CD, Kubernetes, Helm, IaC et monitoring.
- **Automatiser** la chaîne complète « commit → production » sans étape manuelle cachée.
- **Garantir la reproductibilité** : tout se recrée à partir du dépôt.
- **Documenter et démontrer** une solution DevOps de manière professionnelle.
- **Vous auto-évaluer** à l'aide d'une grille de critères précise.

## Contenu

| # | Leçon | Thèmes |

| 01 | [Consignes du projet](01-consignes.md) | Objectif, portée, contraintes, modalités, échéancier, étapes |
| 02 | [Livrables attendus](02-livrables.md) | Dépôt Git, Dockerfile, pipeline, K8s/Helm, IaC, monitoring, doc, démo |
| 03 | [Critères d'évaluation](03-criteres-evaluation.md) | Grille détaillée (barème /100), niveaux de réussite, conseils |

## Format des leçons

Chaque leçon est autonome et suit la même structure pédagogique :

- une **table des matières** cliquable ;
- des **sections repliables** (`<details>`) avec diagrammes **Mermaid** ;
- un **quiz** corrigé (solutions repliables) ;
- une **pratique** avec correction détaillée ;
- une **synthèse** des points à retenir.
